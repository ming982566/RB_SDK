# RaceBear SDK 公共接口缺口审计

审计日期：2026-07-14

## 审计基线

- 标准第三方头：`include/RaceBearSDK.h`
- 自家前端适配头：`SimRacebear/module/RB_SDK/RBSDK/RaceBearSDK.h`
- 实际调用：`SimRacebear/**/*.xc`
- 第三方文档：`docs/RaceBearSDK_API.md`

0.4.3 已补齐动态滤波、输出实例、运动增强、震动、风感和安全带的主要结构体配置流程，并修复全新 `AppDataName` 无法初始化的问题。

0.4.4 已补齐游戏安装与遥测 Source 配置、功能插件目录与动作、平台选择与数值诊断、手动平台测试，并在主要 Save/Apply 入口增加基础参数校验。

0.4.5 修复 CAN 最新值发送和动态输入回读，并公开按 DOF 划分的动态输入效果候选目录，避免第三方前端把完整遥测目录错误展示为可添加通道。

0.4.6 将授权限制统一收口到计算链、运行状态和受保护命令入口。未授权时保留配置与原始遥测能力，屏蔽专有计算结果并安全断开输出。

0.4.7 隐藏历史 SimTools 公共目录项，同时保留内部槽位与索引兼容性。

## 已完成公开

### P0 游戏和遥测源

第三方前端现在可以完成：

- 刷新并读取游戏安装状态、路径和运行状态。
- 按游戏配置创建或重连 Source。
- 自动配置当前游戏、检查游戏侧配置、安装必要插件。
- 通过 `game.restoreLast` Command 恢复上次游戏。

`game.autoDetect`、`game.launch`、`game.terminate` 和 `game.restoreLast` 使用 Command，没有再公开功能重复的细粒度函数。WT901 等特定硬件 Source 命令继续留在自家适配层。

### 暂缓公开：CAN

当前第三方 SDK 只保留 `RB_RuntimeState::Can` 状态摘要和 CAN 输出类型所需的基础配置。电机参数、固件更新、故障恢复和急停流程暂不作为完整第三方能力公开，继续由 SimRacebear 私有适配层维护。

### 功能插件

标准 SDK 已公开结构体目录、刷新、后台运行态启动和插件前端启动。绘制回调仍保留在自家前端层。

### 暂缓公开：LAN 控制

LAN 控制当前没有完整第三方发布计划，标准 SDK 文档不展开设备发现、命令发送和远程关机细节。后端实现继续保留给 SimRacebear，自定义局域网控制由第三方宿主自行实现。

### 手动平台测试和诊断

当前平台索引、DOF 可达性、预览限制、手动测试启用/输入/复位均已公开。真实绘制仍由宿主实现。主要配置入口会拒绝数组越界、非法端口、NaN/Inf、无效百分比和错误速度范围。

## 后续可选补充

### P1 配置管理补充

- 提供完整动态默认配置，而不是要求客户从全零结构体创建。
- 为动态滤波和运动增强分别增加配置重命名，保持两个独立目录各自可管理。
- 为震动增加独立的配置目录枚举、创建、删除和重命名接口；当前震动内容独立，但 `ByIndex` 接口仍按动态配置名关联。
- 增加震动手动测试输出和每效果运行状态。

## 不进入标准 SDK

以下能力继续留在 SimRacebear/炫彩适配层，避免把自家 UI 约束强加给第三方：

- `RB_UI_*` 语言包、控件文本和窗口辅助函数。
- 平台、滤波和插件状态的炫彩绘制包装。
- 自家服务器状态按钮、托盘菜单和页面导航逻辑。
- 日志文件界面和保留天数 UI；标准 SDK 保留 `RB_Log_Read()`，日志文件由宿主保存。

以下旧细粒度接口也不需要重复公开：

- `RB_Game_GetCount()`：使用 `RB_Game_GetCatalog()`。
- `RB_Output_GetAvailableKeyCount()`：使用 `RB_Output_GetCatalog()`。
- `RB_Log_GetCount()`、`RB_Log_GetLastType()`：循环调用 `RB_Log_Read()`。
- `RB_Log_Clear()`：使用 `logs.clear` Command。

CAN 和 LAN 不列入当前第三方 SDK 完整化计划，后续只有在明确决定对外开放时再单独设计公共接口。
